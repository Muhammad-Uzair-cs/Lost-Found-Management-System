✅ 3NF Normalization Proof
1NF — First Normal Form
  • All attributes hold atomic values
  • No repeating groups (categories, locations in separate tables)
  • Each table has a primary key
2NF — Second Normal Form
  • All in 1NF
  • No partial dependencies (building/room stored in locations, not duplicated in items)
  • Category name not in items table
3NF — Third Normal Form
  • All in 2NF
  • No transitive dependencies
  • User details not stored in items (only FK)
  • Location coordinates not in items table
